# 휘발성 개인정보 보호 시스템 (Volatile Privacy Protection System) 아키텍처

## 1. 개요 (Overview)
MZ세대 유저들이 가장 민감하게 생각하는 **생년월일 및 태어난 시간(DOB & TOB)** 등 개인 식별 정보를 영구적으로 보존하지 않는 'Zero-Retention' 아키텍처를 설계합니다.

## 2. 데이터 처리 파이프라인 (Data Flow Pipeline)
사용자의 민감 데이터는 서버의 디스크(DB)에 닿기 전에 즉시 파기되거나, 암호화된 상태로 메모리에서만 연산된 후 증발합니다.

### 2.1. Stateless Token & JWT 연동
- 사용자가 생년월일을 입력하면 프론트엔드에서 즉시 비대칭 키(RSA-2048)로 암호화합니다.
- 백엔드(FastAPI)는 메모리 상에서 이를 복호화하여 **'진태양시 보정 엔진'**에 통과시키고 사주 8글자(명식, Bazi)를 추출합니다.
- 변환된 사주 8글자는 개인을 특정할 수 없는 '명리 데이터 구조'로 취급하여 DB에 해시 처리되어 저장되거나, 사용자의 JWT Payload에 담아 클라이언트로 넘깁니다.
- **연산 직후 메모리에서 생년월일 원본 데이터 파기.**

### 2.2. Redis-based TTL (Time-To-Live) 캐싱
- 세션 유지 및 단기 연산을 위해 원본 데이터가 필요할 경우, RDBMS가 아닌 **Redis 캐시 메모리**를 사용합니다.
- TTL을 10분(600초)으로 강제 설정하여, 처리가 지연되거나 비정상 접근이 발생하더라도 10분 후 자동 소멸(Eviction)되게끔 설계합니다.

## 3. 설명 가능한 AI (XAI) 도입을 통한 신뢰(Trust) 확보
- 단순히 '운세가 이렇습니다'라고 결과를 주면 사용자는 '내 데이터를 몰래 보관해서 템플릿에 맞춰 주는건가?' 의심할 수 있습니다.
- 답변을 생성할 때, AI가 **"당신의 입력 데이터는 연산 후 안전하게 삭제되었으며, 오직 '목(木)' 기운이 강한 사주의 구조적 특징(식신 등)만을 바탕으로 조언을 생성했습니다."**라는 데이터 흐름 관련 안내를 투명하게 제공하여 UI/UX에 신뢰감을 더합니다.
